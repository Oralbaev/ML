# SIS-3: Iris Classification System

An end-to-end machine learning project that trains a RandomForest classifier on the Iris dataset, tracks experiments with MLflow, serves predictions via FastAPI, and provides an interactive Streamlit UI.

---

## How this extends Practical Task 6

Practical Task 6 was a minimal ML service: a single `train.py` that saved a model to disk and a `main.py` FastAPI that loaded it and served `/predict`.

SIS-3 builds on top of that foundation by adding:

| Feature | Practical Task 6 | SIS-3 |
|---|---|---|
| FastAPI `/predict` endpoint | Yes | Yes (extended with confidence score) |
| Streamlit frontend | No | Yes |
| MLflow experiment tracking | No | Yes |
| MLflow Model Registry | No | Yes |
| Structured project layout | No | Yes (`app/`, `frontend/`, `ml/`) |
| Multi-service docker-compose | No | Yes (MLflow + API + Streamlit) |

## Running with Docker (recommended)

### Step 1 — Start all services

```bash
docker-compose up --build
```

This starts three services:

| Service | URL |
|---|---|
| MLflow UI | http://localhost:5000 |
| FastAPI | http://localhost:8000 |
| Streamlit | http://localhost:8501 |

### Step 2 — Train the model

The model must be trained before the API can serve predictions. Run the training script from the project root:

```bash
pip install -r requirements.txt
python ml/train.py
```

The script connects to MLflow at `http://localhost:5000` (the service started by docker-compose), logs the experiment, and saves `models/model.joblib` locally.

### Step 3 — Restart the API

If the API container started before the model was trained, restart it so it picks up the new model file:

```bash
docker-compose restart api
```

## Running locally (without Docker)

### 1. Install dependencies

```bash
pip install -r requirements.txt
```

### 2. Start MLflow

```bash
mlflow server --host 0.0.0.0 --port 5000
```

MLflow UI will be available at http://localhost:5000.

### 3. Train the model

```bash
python ml/train.py
```

This will:
- Create the `iris_classification` experiment in MLflow
- Log parameters, accuracy, and F1-score
- Register the model as `iris_random_forest` in the MLflow Model Registry
- Save `models/model.joblib` for the API to load

### 4. Start FastAPI

```bash
uvicorn app.main:app --reload --port 8000
```

API will be available at http://localhost:8000.  
Interactive docs (Swagger UI): http://localhost:8000/docs

### 5. Start Streamlit

```bash
streamlit run frontend/streamlit_app.py
```

UI will be available at http://localhost:8501.

## API Reference

### GET /

Health check.

```bash
curl http://localhost:8000/
```

Response:

```json
{"message": "Iris Classification API is running"}
```

### GET /health

Returns whether the model is loaded.

```bash
curl http://localhost:8000/health
```

Response:

```json
{"status": "ok", "model_loaded": true}
```

### POST /predict

Predict the Iris species from flower measurements.

```bash
curl -X POST http://localhost:8000/predict \
  -H "Content-Type: application/json" \
  -d '{
    "sepal_length": 5.1,
    "sepal_width": 3.5,
    "petal_length": 1.4,
    "petal_width": 0.2
  }'
```

Response:

```json
{
  "prediction": 0,
  "class_name": "setosa",
  "confidence": 1.0
}
```

**Request fields:**

| Field | Type | Description |
|---|---|---|
| `sepal_length` | float | Sepal length in cm |
| `sepal_width` | float | Sepal width in cm |
| `petal_length` | float | Petal length in cm |
| `petal_width` | float | Petal width in cm |

**Response fields:**

| Field | Type | Description |
|---|---|---|
| `prediction` | int | Class index: 0, 1, or 2 |
| `class_name` | string | `"setosa"`, `"versicolor"`, or `"virginica"` |
| `confidence` | float | Model confidence (0.0–1.0) |

## MLflow Details

After running `python ml/train.py`, open http://localhost:5000 to see:

**Experiments tab:**
- Experiment name: `iris_classification`
- Run name: `random_forest_baseline`
- Logged parameters: `n_estimators`, `max_depth`, `min_samples_split`, `random_state`
- Logged metrics: `accuracy`, `f1_score`
- Logged artifact: serialized scikit-learn model

**Model Registry tab:**
- Registered model name: `iris_random_forest`
- Each training run creates a new version
